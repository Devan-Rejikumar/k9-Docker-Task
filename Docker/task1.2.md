1. Why did the NGINX container keep running while the busybox container exited?
The NGINX container keeps running because it runs a foreground process, which is the NGINX server (nginx -g daemon off;). In Docker, a container stays alive as long as its main process (PID 1) is running. Since NGINX continuously listens for incoming requests, the process does not exit, so the container remains running.
In contrast, the busybox container executed a short-lived command that completed immediately, causing its main process to exit, and therefore the container stopped.

2. What does detached mode mean in Docker?
Detached mode in Docker is enabled using the -d flag. It runs the container in the background and frees up the terminal, allowing the user to continue executing other commands. In contrast, without detached mode, the container runs in the foreground and occupies the terminal.

3. What does -p 8080:80 mean exactly?
The -p 8080:80 option is used for port mapping in Docker. The left side, 8080, is the host machine’s port, and the right side, 80, is the container’s port. It means that any request coming to port 8080 on the host is forwarded to port 80 inside the container where the application is running.

4. Difference between host port and container port
The host port is the port on the local machine that receives external traffic, while the container port is the port inside the container where the application is running. These two are connected using port mapping, which forwards requests from the host port to the container port.

5. How did traffic from the browser reach the container?
Traffic from the browser reaches the container through port mapping. When a request is sent to the host machine on port 8080, Docker forwards that request to port 80 inside the container, where NGINX is running. NGINX processes the request and sends the response back to the browser.

6. What is the purpose of docker logs?
docker logs is used to view the output generated by a container, including application logs, errors, and events. It helps in understanding what is happening inside the container and is mainly used for debugging and monitoring.

7. What is the purpose of docker inspect?
docker inspect is used to retrieve detailed information about a container’s configuration and runtime state. It provides data such as container status, network settings, IP address, port mappings, and environment configuration, which helps in debugging and verification.

8. What is the purpose of docker top?
docker top is used to view the running processes inside a container. It helps verify which processes are active and ensures that the expected application is actually running inside the container.

9. What is the purpose of docker exec?
docker exec is used to run commands inside a running container. It allows us to interact with the container’s environment, inspect files, and perform debugging by executing commands within the container.

10. What is the purpose of docker stats?
docker stats is used to monitor the real-time resource usage of a container, including CPU, memory, network I/O, and the number of processes. It helps in understanding performance and detecting resource-related issues.

11. What did you observe about CPU and memory usage?
I observed that the CPU usage was almost 0% and the memory usage was very low, around 10 MB. This indicates that the NGINX container is lightweight and was mostly idle, as it was only handling a few requests.

12. Why is checking resource usage useful in production?
Checking resource usage is important in production to monitor the performance of applications. It helps identify issues like high CPU usage, memory leaks, or sudden traffic spikes, which can lead to slowdowns or crashes. By monitoring these metrics, we can take action before the system fails.

13. What happened when you ran docker stop?
When I ran docker stop, Docker sent a signal to stop the main process inside the container, which is the NGINX server. Once the main process stopped, the container also stopped because a container’s lifecycle is tied to its main process.

14. Why does removing the container not remove the image?
Removing a container does not remove the image because containers and images have separate lifecycles. A container is a running instance of an image, and Docker keeps the image so it can be reused to create new containers without downloading it again.
